## 变更内容

本 PR 主要围绕“使用 Fluent 2 美化媒体管理器外观”展开，并补齐在改造过程中发现的启动与交互问题：

- 引入 **Fluent 2**（`@fluentui/react-components`），在渲染入口接入 `FluentProvider`，新增统一主题配置（品牌色保持 `#005FB8`）。
- 重构核心界面组件样式与结构：`App`、`TitleBar`、`SearchBar`、`Sidebar`、`TabBar`、`Toast`，并统一全局背景、字体与滚动条风格。
- 统一右键菜单视觉语言：将彩色 emoji 菜单项替换为白灰线性图标，统一菜单背景、边框、悬浮态与文字层级（覆盖媒体网格、播放器、侧栏空白处、Tab 菜单）。
- 修复启动期问题：增加原生依赖重建脚本（`postinstall` / `rebuild:native`），解决 `better-sqlite3` 在 Electron 下 ABI 不匹配导致的数据库初始化失败和 IPC 连锁报错。
- 清理仓库运行时数据管理：将 `data/` 目录加入 `.gitignore`，并移除历史上误跟踪的运行时缩略图与数据库临时文件。
- 根据后续反馈，**移除 Tab 栏“新建播放列表”按钮**，避免与“固定侧边栏”按钮产生冲突，同时回退不美观的临时布局方案。

## 为什么做这些修改

- 任务目标是把媒体管理器界面升级到 Fluent 2，提升整体观感和一致性。
- 在实际运行中暴露出启动报错（原生模块版本不匹配）与样式告警（Fluent/Griffel class 与样式写法），需要一并修复以保证可用性。
- 右键菜单此前存在明显风格割裂（彩色图标/高饱和强调色），与整体白灰线框风格不统一，因此做了系统化统一。
- “新建播放列表”按钮与侧栏固定按钮存在位置冲突，最终按产品反馈直接移除该按钮以保证界面简洁稳定。

## 重要实现细节

- 新增主题文件：`src/renderer/theme/fluentTheme.ts`。
- Fluent 入口注入：`src/renderer/main.tsx`。
- 启动与原生依赖相关脚本更新：`package.json` 新增 `postinstall` 与 `rebuild:native`。
- 交互层面保留原有核心逻辑（播放、标签、右键操作等），重点调整 UI 呈现与可用性。
- 当前分支包含 `dist/main/*` 的构建产物更新，与本次启动修复联动提交。
